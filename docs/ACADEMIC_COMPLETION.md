# Academic Completion Criteria

This document defines the review standard for the risk-aware UAV return-to-home project.

## Research-grade objective

The project should formulate return-to-home as a probabilistic safety decision problem for energy-constrained UAVs. Completion requires a reproducible comparison between deterministic battery-threshold logic and risk-aware policies under battery uncertainty, battery health degradation, and wind disturbances.

## Minimum completion standard

A robotics reviewer should be able to verify:

1. The probabilistic safe-return model is mathematically defined.
2. Deterministic and risk-aware policies are implemented under a common interface.
3. Monte Carlo sample count, random seeds, and scenario parameters are configurable.
4. Battery state-of-charge uncertainty and battery-health degradation are modeled explicitly.
5. Wind disturbance experiments include static and dynamic/gust conditions.
6. Evaluation includes safe return, mission completion, failure rate, trigger timing, energy margin, and runtime.
7. Ablation studies quantify the trade-off between Monte Carlo samples and decision quality.
8. Plots and tables are generated by scripts.
9. ROS 2 or robotic-system integration is framed honestly as integration path or implemented module.
10. Reported results are tied to command-line scripts and output files.

## Evaluation protocol

| Experiment | Evidence expected |
|---|---|
| Main policy comparison | aggregate metrics over fixed trials |
| Battery-aging study | performance versus battery-health level |
| Wind-gust study | trigger/failure behavior under dynamic wind |
| MC ablation | runtime and decision quality versus sample count |
| Timeline analysis | representative decision trajectory |
| Robustness discussion | failure modes and safety-efficiency trade-off |

## Definition of done

The project is review-ready when a clean checkout can run the documented experiments and regenerate the reported CSV files, plots, and tables.

## Research interpretation

A strong final discussion should explain the safety-efficiency trade-off. A risk-aware policy may return earlier and reduce mission completion, but that behavior is justified if it reduces unrecoverable low-energy failure or provides a calibrated estimate of return feasibility.
